**File Pattern:** `issue-XXXXX-analysis.pdf`

**Example:**

### üí° Solution PDFs (Only When Solutions Found)
**File Pattern:** `solution-X-issue-XXXXX.pdf`

These are **ONLY** generated when Stack Overflow solutions are found for the issue.

**Example with Solutions:**
```bash

npm run auto-solutions https://github.com/facebook/react/issues/XXXXX
```

**Result (if solutions found):**
```

‚úÖ solution-1-issue-XXXXX.pdf (REAL - from Stack Overflow)
‚úÖ solution-2-issue-XXXXX.pdf (REAL - from Stack Overflow)
‚úÖ solution-3-issue-XXXXX.pdf (REAL - from Stack Overflow)
```

**Result (if NO solutions found):**
```
‚úÖ issue-XXXXX-analysis.pdf (REAL)
‚ùå No solution PDFs (because no Stack Overflow solutions exist)
```


## üéØ How to Get Real Solution PDFs

### Option 1: Find Issues with Solutions

Use comprehensive analysis to find issues that have Stack Overflow solutions:

```bash
# Analyze a repository
npm run comprehensive-analyze facebook react 50

# Check which issues have solutions
cat facebook-react-comprehensive-50.json | jq '.[] | select(.stackOverflowSolutions | length > 0) | {issueNumber, solutionCount: (.stackOverflowSolutions | length)}'

# Generate PDFs for those issues
npm run auto-solutions https://github.com/facebook/react/issues/XXXXX
```

### Option 2: Try Common Bug Issues

Bug issues are more likely to have Stack Overflow solutions:

```bash
# Try a common React error
npm run auto-solutions https://github.com/facebook/react/issues/14099

# Try a Flutter build error
npm run auto-solutions https://github.com/flutter/flutter/issues/XXXXX

# Try a VS Code extension issue
npm run auto-solutions https://github.com/microsoft/vscode/issues/XXXXX
```

### Option 3: Search Stack Overflow First

```bash
# 1. Find a Stack Overflow question
# 2. Look for related GitHub issues in the question
# 3. Run auto-solutions on that GitHub issue
npm run auto-solutions <github-issue-url>
```
